Welcome to OAIZer-DB

The easy way to export your relational database via OAI protocols....

Introduction

Relational Databases are great to implement your document model and manage your data but their export functionalities are limited to CSV or SQL exports that reflect the structure of the relations defined in the database. What if you want to export portions of your information space as (XML or RDF) "export packages" of interlinked information objects via OAI-PMH and OAI-ORE protocols?

Instead of implementing your own OAI-PMH and OAI-ORE publishers from scratch, you can use OAIzer-DB!

OAIzer-DB connects to any JDBC-compatible RDBMS and provides you with graphical user interfaces for the definition of the structure of export packages and the selection of the desired OAI protocol. Once structure and protocol are provided, the tool deploys an OAI HTTP address and installs the code required to respond to requests by dynamically building export packages according to the given structure.

Step 1: Load the document model from the database

Specify the parameters to connect to your database

Click on Connections --> Create Connection from the menu and fill in the requested fields, then click on the button "Try Connection" to check if the inserted information are correct.

Creating a new connection to a relational database

Load the database schema

Click on ER-schema --> Load ER-schema

OAIzer will connect to your database with the provided parameters to get information about the database schema from the Relation Catalog. The resulting graph document model is then visualized

The graph document model contains one entity for each table (using the table name) and one one-to-many relationship for each foreign key (using the attribute name).

ER Schema created by OAIzer based on the information from the Relation Catalog

Define the graph document model from the ER-schema

Some entities are "worth" further human analysis since they may be representing many-to-many relationships. Such entities are those with two (or more) outgoing one-to-many relationships, hence candidate to become themselves many-to-many relationships. In the example, this is the case for the entities "uses" and "cites".

Those entity are logically representing relationships. If you click on them, you can tell OAIzer that those entities are in fact relationships.

Create the graph document model from the ER Schema: selecting tables that are many-to-many relationships

Save your graph document model

Click on ER-schema --> Save ER-schema to save your changes

Step 2: Define your views

Views define the structure of the export packages as rooted trees within the document model.

  1. Click on XXX --> YYY
  2. Give a name to the view you are going to create
  3. Select the root entity of the view
  4. Start navigating the graph document model from the root entity by clicking on relationships. When you select a relationship you are including into the view that relationship and its target entity.

For each entity it is possible to specify the subset of attributes to be included in the export packages.

Example of a view

Step 3: Save and export your view

Click on XXX --> YYY to save your view.

Click on XXX --> YYY to select the desired OAI protocol and the modality (materialization or on-demand mode). Currently, only OAI-ORE on-demand mode is supported.

OAIzer deploys an OAI web endpoint you can use to access your export package:

  • OAI-ORE endpoint: http://host/view_name/ore.
  • To access an aggregation: http://host/view_name/ore/aggregation/record_identifier.
  • To access an aggregated resource: http://host/view_name/ore/resource/record_identifier

where record_identifier is the db primary key of the record you want to access (aggregations will be available only for records of the root entity).
